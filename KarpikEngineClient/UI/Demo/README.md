# KarpikEngine UI System Demo

Комплексное демо для демонстрации всех возможностей UI системы KarpikEngine.

## Структура демо

### 1. Базовые элементы (`UIDemo.cs`)
- **Label** - текстовые элементы с различными стилями выравнивания
- **Button** - интерактивные кнопки с событиями клика
- **Toast** - уведомления различных типов (Info, Warning, Error)

### 2. Layout контейнеры
- **HBox** - горизонтальное размещение элементов
- **VBox** - вертикальное размещение элементов  
- **Grid** - табличное размещение с настраиваемыми колонками и строками
- **Nested Layouts** - вложенные layout'ы для сложных интерфейсов

### 3. Специализированные контейнеры
- **ScrollView** - прокручиваемый контейнер с полосами прокрутки
- **Card** - карточки с тенью и скругленными углами
- **Panel** - панели с возможностью сворачивания
- **Foldout** - складывающиеся секции с анимацией

### 4. Интерактивные элементы (`InteractiveDemo.cs`)
- **Hover Effects** - эффекты при наведении мыши
- **Click Handlers** - обработка кликов и событий
- **Focus Management** - управление фокусом элементов
- **Animation Controls** - кнопки управления анимациями

### 5. Анимации и эффекты
- **Fade In/Out** - плавное появление/исчезновение
- **Scale** - масштабирование элементов
- **Move** - перемещение с анимацией
- **Color Transitions** - плавная смена цветов
- **Shake** - эффект тряски
- **Pulse** - пульсация элементов

### 6. Стилизация (`DemoStyles.cs`)
- **Color Schemes** - предопределенные цветовые схемы
- **Border Radius** - различные варианты скругления углов
- **Padding & Margin** - отступы и поля
- **Typography** - настройки шрифтов и текста
- **Hover States** - стили для состояний наведения

## Использование

### Быстрый запуск
```csharp
// В методе CreateNewUI() класса Client
var demo = DemoLauncher.QuickDemo();
_uiManager.SetRoot(demo);
```

### Полное демо
```csharp
// Создание комплексного демо со всеми возможностями
var comprehensiveDemo = DemoLauncher.CreateDemo(DemoLauncher.DemoType.Comprehensive);
_uiManager.SetRoot(comprehensiveDemo);
```

### Специфичные демо
```csharp
// Только базовые элементы
var basicDemo = DemoLauncher.CreateDemo(DemoLauncher.DemoType.BasicElements);

// Только layout системы
var layoutDemo = DemoLauncher.CreateDemo(DemoLauncher.DemoType.Layouts);

// Интерактивные элементы и анимации
var interactiveDemo = DemoLauncher.CreateDemo(DemoLauncher.DemoType.Interactive);
```

## Файлы демо

- **`UIDemo.cs`** - Основное демо с базовыми элементами и контейнерами
- **`InteractiveDemo.cs`** - Интерактивные элементы и анимации
- **`ComprehensiveDemo.cs`** - Полное демо с навигацией между секциями
- **`DemoStyles.cs`** - Предопределенные стили для демо
- **`DemoLauncher.cs`** - Утилита для запуска различных типов демо

## Возможности демо

### Навигация
- Переключение между различными секциями демо
- Информационная панель с FPS и количеством элементов
- Интуитивный интерфейс навигации

### Интерактивность
- Клики по элементам с визуальной обратной связью
- Hover эффекты для улучшения UX
- Анимации при взаимодействии с элементами

### Производительность
- Тестирование с большим количеством элементов
- Мониторинг производительности рендеринга
- Оптимизация для различных сценариев использования

### Стилизация
- Демонстрация различных цветовых схем
- Примеры кастомизации внешнего вида
- Responsive дизайн для разных размеров экрана

## Расширение демо

Для добавления новых элементов в демо:

1. Создайте новый элемент в соответствующем файле
2. Добавьте стили в `DemoStyles.cs`
3. Интегрируйте в `ComprehensiveDemo.cs` для навигации
4. Обновите `DemoLauncher.cs` при необходимости

## Технические детали

- Использует архитектуру VisualElement как базовый класс
- Поддерживает вложенные layout'ы любой сложности
- Оптимизировано для производительности с большим количеством элементов
- Полная поддержка событий мыши и клавиатуры
- Интеграция с системой анимаций KarpikEngine

## Требования

- KarpikEngine UI Core система
- Raylib для рендеринга
- .NET для логики и событий

Демо предоставляет полный обзор возможностей UI системы и служит как документацией, так и инструментом для тестирования новых функций.
